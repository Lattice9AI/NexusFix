# Roadmap

## Near Term

- Keep benchmark claims reproducible and current with public baselines
- Strengthen cross-platform build and release polish
- Improve documentation around package consumption and deployment tradeoffs
- Expand public guidance for enabling or disabling optional acceleration features

## Mid Term

- Improve downstream integration examples for packaged and embedded deployments
- Expand public benchmark comparisons across feature configurations
- Refine portability guidance for SIMD and non-Linux transport configurations

## Longer Term

- Continue documenting architecture decisions behind latency-sensitive subsystems
- Expand public release discipline around upgrade notes and compatibility expectations
